# UNSU Platform - 개발 인수인계서 (HANDOVER.md)

## 🕒 업데이트 일시
- **2026년 06월 24일** (작업 종료 시점 기준)

## ✅ 현재 상태 및 오늘 완료한 주요 작업

### 1. 운영계 환경(Production) 연동 이슈 원인 분석 및 가이드
- **이슈:** `daily_lucky_cards`, `recommended_courses` DB에 데이터가 쌓이지 않고 Mock 데이터만 나오는 현상.
- **원인:** 로컬(`.env`)과 달리 운영계(Render)의 환경변수가 비어 있어서 발생.
- **조치:** Render.com 대시보드 환경변수에 로컬의 `DATABASE_URL` 및 API 키를 모두 추가하고 재배포하면 데이터가 적재됨을 가이드.

### 2. 공공데이터 API 키 오용 문제 해결 (중요)
- **이슈:** `DATA_GO_KR_API_KEY`에 입력된 UUID 키가 작동하지 않고 401(Unauthorized) 에러와 함께 Mock 데이터를 반환하던 현상.
- **원인:** 제공해주신 키는 **"공공데이터포털(data.go.kr)"** 키가 아니라 이름이 유사한 **"문화 공공데이터광장(culture.go.kr)"**의 서비스 키였음.
- **조치:** 올바른 공공데이터포털 회원가입 및 재발급 절차 가이드 제공. 추가로 새로 발급된 "문화 공공데이터광장" API를 EventMonitor 등 백엔드에서 호출할 수 있도록 문화예술공연 연동 스크립트 작성 준비.

### 3. 프론트엔드 네비게이션 및 UX 디테일 버그 픽스
- **[인트로] 스플래시 화면 흐름 개선:** 최초 접속 스플래시 화면에서 "시작하기" 클릭 시, 유저가 어느 페이지로 접속했든 무조건 메인 대시보드인 **'루틴'(`/`)** 화면으로 강제 이동하도록 `AppLayout.tsx` 라우팅 수정 완료.
- **[루틴] 날씨 위치 표기 상세화:** 기사님의 날씨 정보가 "서울특별시"로만 표기되던 문제를 수정. 클라이언트에서 넘겨준 GPS 위/경도를 백엔드(`server.ts`)가 정상적으로 수신하고, `getRegionFromCoords` 함수를 태워 "서울특별시 강남구"와 같이 **'구' 단위까지 정밀하게 표기**되도록 로직 개선 완료.

### 4. 긴급 화장실 탐색 API 위치 기반 검증 (분석 완료)
- **분석 결과:** 현재 화장실 탐색은 진짜 실시간 위치 기반이 아닌 **Mock(가짜 랜덤) 거리**를 사용하고 있음. 
- **원인:** 연동된 공공데이터 화장실 API(`info_v2`)가 위/경도 기반 거리 검색을 지원하지 않아 전국 데이터 중 무작위로 1페이지를 불러와 표기함.
- **해결 방안 제안:** 추후 카카오맵 장소 검색 API(키워드 '화장실') 연동으로 전면 개편하거나 전국 DB를 통째로 이관해야 함을 인수인계함.
