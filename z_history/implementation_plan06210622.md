## 🛠️ [신규 제안] API 부하 분산 및 이벤트 관리 BO 구축 계획

고객님께서 지적해주신 **"시스템 부하 문제 (모두가 다 실시간으로 통신하는가?)"**는 매우 예리하고 중요한 포인트입니다.
현재 상태로는 사용자가 늘어날 경우 공공데이터 API의 일일 호출 제한(Rate Limit)에 걸리거나 서버가 다운될 수 있습니다.

### 1. API 서버 사이드 캐싱 (부하 해결책)
모든 클라이언트가 실시간으로 외부 API를 찌르지 않도록 **Node.js 서버단에 인메모리 캐싱(Cache)**을 적용합니다.
*   **방식**: 서버가 외부 API에서 데이터를 가져오면 1~5분간 메모리에 저장해두고, 해당 시간 내의 클라이언트 요청은 캐시된 데이터를 즉시 반환합니다.
*   **효과**: 1만 명이 접속해도 외부 API 호출은 1~5분에 딱 1번만 발생하므로, 시스템 부하가 99% 이상 감소합니다.

### 2. 이벤트 관리 BO 화면 (`EventMonitor.tsx`)
10개의 문화행사 소스에서 통합된 데이터를 백오피스에서 한눈에 관리할 수 있는 화면입니다.
*   **주요 기능**: 카테고리별(콘서트, 스포츠 등)/지역별 필터링, 예상 관람객 수에 따른 '핫존(Surge)' 발생 여부 시각화 등.
