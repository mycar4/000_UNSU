# UNSU 플랫폼 실데이터베이스 및 REST API 통합 연동 완료 리포트 (v0.7)

본 문서는 UNSU 플랫폼의 프론트엔드 모바일 클라이언트(FO)와 본사 백오피스(BO)가 로컬 모의 상태(`localStorage`)를 탈피하여, 실제 백엔드 API 서버(Express) 및 데이터베이스(PostgreSQL/Supabase)와 완전 연동(E2E Integration)되고 사용자의 피드백을 수렴하여 보안 및 편의 기능을 추가 구현한 최종 개발 완료 보고서입니다.

---

## 1. 보안 및 개인정보 관리 고도화

기사의 주민등록번호/홈택스 식별키 등 민감정보(PII)의 안전한 취급을 위해 강력한 암호화 메커니즘을 가동했습니다.

### ① 양방향 AES-256-GCM 암호화 연동
*   기사가 입력한 국세청 홈택스 ID(`hometax_id`)는 백엔드 DB 저장 시 무작위 12바이트 IV와 인증 태그(Auth Tag)를 사용하여 암호화되며, `iv:authTag:ciphertext` 포맷으로 데이터베이스에 안전하게 보관됩니다. ([crypto.ts](file:///d:/000_UNSU/api/src/utils/crypto.ts))
*   백오피스(BO) 또는 모바일 클라이언트(FO)에서 정상 권한으로 조회 시, DB 래퍼 단계에서 복호화 레이어를 거쳐 복호화된 정보로 변환되어 전달됩니다. ([db.ts](file:///d:/000_UNSU/api/src/utils/db.ts))

### ② Super Admin 제외 일반 관리자 대상 PII 마스킹 가이드 적용
*   보안성 표준 가이드를 지키기 위해, 백오피스(BO) 기사 정보 관리 화면에서 로그인 세션의 역할(`admin_role`)을 확인합니다.
*   최고관리자(`Super Admin`) 역할을 갖지 않은 일반 관리자(Manager, Auditor 등)가 기사 정보를 조회할 경우, 홈택스 ID는 앞 3자리만 노출하고 뒷부분은 전체 `*` 처리하는 마스킹 가이드를 엄격히 가동합니다.
*   더불어, 수정용 폼 에디터에서도 `Super Admin` 이외 등급은 국세청 홈택스 ID 입력란이 **읽기 전용(`readOnly`) 및 비활성화(`disabled`)** 처리되어 임의 변경이 원천 차단됩니다.

### ③ 단방향 SHA-256 해시를 활용한 탈퇴 관리 및 3일 재가입 제한
*   기사가 회원 탈퇴(`POST /api/drivers/:id/withdraw`)를 요청할 때, 개인정보 보호법에 의거하여 기사의 프로필 데이터는 즉시 영구 삭제(CASCADING DELETE) 처리됩니다.
*   다만, **탈퇴 후 3일간 재가입 금지** 정책을 준수하기 위해 주민번호(홈택스 ID)를 복원 불가능한 단방향 SHA-256 알고리즘으로 해싱하여 고유 식별 해시값만을 `withdrawn_drivers` 테이블에 보존합니다.
*   신규 온보딩 가입 시도 시, 입력받은 홈택스 ID를 해싱하여 해당 락 테이블에 3일 이내에 등록된 이력이 존재하면 가입이 차단되며, 가입이 가능해지는 정확한 해제 시각을 친절하게 안내합니다.

## 2. 프론트엔드(FO) API 및 사용자 경험(UX) 개선 사항

사용자의 검토 의견과 피드백을 반영하여 아래와 같이 UI 및 기능적 요건들을 개선했습니다.

### ① 오늘 날짜 및 요일 상단 브리핑 영역 추가 및 가독성 강화
*   기사님들의 하루 루틴 페이지인 [GillogPage.tsx](file:///d:/000_UNSU/fo/src/pages/GillogPage.tsx) 상단 헤더에 오늘 날짜(예: `2026년 6월 20일 토요일`)를 컴팩트하고 세련된 텍스트 서체(`text-xs sm:text-sm font-semibold text-gold bg-gold/5 border border-gold/20 px-2.5 py-1 rounded-lg`)로 적용했습니다.
*   또한 상단 설명 멘트(출근 전 가볍게 확인하세요...) 역시 기존의 거대한 사이즈에서 시인성이 좋은 미니멀 크기(`text-xs sm:text-sm text-muted-foreground/80`)로 줄여 상단 레이아웃을 쾌적하게 확보했습니다.

### ② 흐릿했던 텍스트 라벨 시인성 대폭 개선
*   디자인 일관성으로 인해 다소 흐릿하게 보이던 텍스트 레이블들을 선명하게 보정 완료했습니다.
    *   **"운세 브리핑"**: `text-foreground/90 font-extrabold`와 백그라운드 구분을 주어 가시성 강화.
    *   **"아침 핵심 추천 코스"**: `text-primary font-extrabold` 및 브랜드 배경색을 곁들여 최우선 강조.
    *   **"조회수 1.2만"**: 생존 가이드의 하단 통계가 명료하게 인지되도록 `text-[11px] text-foreground/80 font-bold font-sans`로 재배치.

### ③ 선호 내비게이션(TMAP / 카카오네비) 선택권 및 딥링크 동적 라우팅
*   **프로필 설정 확장**: 온보딩 최초 가입 시 기사가 원하는 기본 내비게이션(티맵 또는 카카오네비)을 선택할 수 있는 컬럼(`navi_preference`)을 추가하고 암호화하여 DB에서 통합 관리합니다.
*   **동적 딥링크**: 오늘의 루틴 추천 코스 진입 시 설정된 내비 선호도에 맞춰 모바일 클라이언트에서 T-MAP 스키마(`tmap://`) 또는 카카오네비 스키마 (`kakaonavi://`)를 스마트하게 선택적으로 분기 호출(Dynamic Dispatching)하게 구성했습니다.

### ④ G-PAN 레이더 오프라인 예외 처리 및 직관적 안내 UI
*   **오프라인 폴백 캐싱**: 통신 불량 또는 외부 API 오류로 핫존 데이터를 긁어오지 못할 경우, 화면이 깨지지 않고 정상 작동하도록 localStorage에 백업된 캐시 정보를 복구해 화면을 가동합니다.
*   **미니멀 경고 배너**: 텍스트 위주의 지루한 경고 대신 직관적인 경고 아이콘(`⚠️ LOCAL CACHE ACTIVE`)을 상단에 띄우고, "네트워크 연결이 불안정하여 마지막 오프라인 캐시로 가동 중입니다."라는 아주 간략하고 명료한 안내 글을 표출합니다.

### ⑤ 영수증 업로드 시 촬영 및 파일 업로드 다중 선택 지원
*   모바일 환경에서 input의 `capture="environment"` 강제 제약 조건을 걷어내어, 실시간 카메라 촬영뿐만 아니라 갤러리나 폰 내부 폴더에 있는 영수증 이미지를 선택하여 유연하게 OCR 등록을 진행할 수 있도록 개선하였습니다.

### ⑥ 전통 명리학 기사 사주(Lucky Card) 등급 판정 로직
*   기사의 생년월일과 운행 당일 날짜의 일진을 해싱 알고리즘으로 조합하여 결정적(Deterministic)으로 연산하여 등급을 산출합니다.
    *   **최상 (BEST)**: 기사의 사주와 오늘 일진이 합(合)을 이루고 천을귀인이 비추어 장거리 재물 운이 최고조인 상태.
    *   **상 (GOOD)**: 상생(相生) 관계로 고단가 호출지(강남/청담) 부근 매칭률이 높은 상태.
    *   **우수 (NORMAL)**: 평온한 상태로 출퇴근길 도심 주요 오더 위주로 균형 잡힌 주행이 유리한 상태.
    *   **평온/안전 (BAD)**: 충(沖)이나 원진(怨嗔)이 있어 주의가 필요한 상태로, 공항 슬롯 진입 등 리스크를 최소화하는 정체 구역 대기를 유도함.
*   기사 정보 노출 시 `"01-01"` 형태의 지루한 표현 대신 `"72년05월"`과 같은 친숙한 형식으로 포맷팅하여 편안한 가독성을 보장합니다.

## 3. 백오피스(BO) 신규 기능 통합 내역

본사 대시보드(BO) 시스템에 감사 추적, 기사 제어 및 API 통합 테스트가 가능한 전문 도구들을 탑재했습니다.

### ① 기사 회원 정보 관리 패널 (`DriverManagement` - /drivers)
*   **기사 프로필 복호화 조회**: 기사 ID(예: `driver-101`)를 검색하면 DB에 안전하게 암호화 보관 중인 홈택스 ID를 양방향 복호화하여 노출합니다. (단, 최고관리자 외 등급 접근 시 마스킹 가이드가 적용됩니다)
*   **마스터 프로필 수정 및 강제 회원 탈퇴**: 생년월일, 출생시간, 영업구분, 내비게이션 선호도, 홈택스 식별 ID를 수정 및 반영할 수 있는 에디터 폼을 제공합니다. 수정 완료 시 즉시 AES-256-GCM 암호화 블록이 저장되고 관리 감사 로그(`DRIVER_UPDATE`)가 생성됩니다. 또한, "기사 강제 탈퇴" 처리 버튼을 제공하여 3일 락 테이블에 안전하게 등록되도록 유도합니다.

### ② 변경 이력 조회 팝업 모달 (`Audit History`) 및 관리자 정보 변경 기능
*   기사 정보 변경 시뿐만 아니라, **관리자 계정 정보(이름, 역할)**를 변경할 때에도 이전 → 이후의 변경 사항을 추출하여 `ADMIN_UPDATE` 감사 로그로 데이터베이스에 로깅하고 있습니다.
*   관리자 모달 내에서 최고관리자(`Super Admin`)의 경우 인라인으로 직접 이름을 수정하거나 역할을 변경하여 저장할 수 있습니다.
*   관리자 목록 우측의 **"이력"** 버튼을 클릭하면, 해당 관리자 계정에 대하여 발생한 신규 생성(`ADMIN_CREATE`), 정보 변경(`ADMIN_UPDATE`), 삭제(`ADMIN_DELETE`) 내역 전체를 상세하게 추적할 수 있는 팝업 모달이 활성화됩니다.
*   또한 전체 시스템 감사 로그 대시보드(/audit-logs)에도 `ADMIN_UPDATE` 액션에 해당하는 이력 조회가 가능하도록 필터 옵션과 인디고 컬러의 배지를 추가 탑재했습니다.
*   기존 기사 정보 수정과 동일하게, BO 화면의 **"변경 이력 보기 (Audit History)"** 버튼을 클릭하면 팝업창이 열려 기사 변경 이력을 볼 수 있는 기능 역시 온전히 연동되어 있습니다.

### ③ 실시간 시스템 감사 로그 대시보드 (`AuditLogsDashboard` - /audit-logs)
*   관리자 추가/삭제, 기사 프로필의 강제 수정, 시스템 로그인 등 중요한 보안 조작이 일어날 때마다 `admin_audit_logs` 테이블에 감사 추적 레코드가 적재됩니다.
*   BO 화면에 통합된 **"감사 로그"** 탭을 통해 관리자는 날짜, 작업자 이메일, 액션 유형별로 필터링하여 시스템 로그를 실시간 모니터링할 수 있습니다.
*   `LOGIN`, `ADMIN_CREATE`, `ADMIN_DELETE`, `DRIVER_UPDATE` 등 성격별 배지(oklch 색상 토큰 반영)로 시각적 가독성을 높였습니다.
*   *주의: 직접 주소창에 `/audit-logs`로 접근 시에도 BO 로그인을 마친 세션 상태에서만 페이지 렌더링이 가동됩니다.*

### ④ Swagger 스타일의 API 테스트베드 (`ApiPlayground` - /api-playground)
*   백엔드에서 제공되는 6대 핵심 통합 API를 대시보드 내부에서 다이렉트로 검증 및 테스트할 수 있는 대화형 콘솔 뷰어입니다.
*   기사 프로필 조회/수정(관리자 헤더 `x-admin-email` 검증 지원), 회원탈퇴, 관리자 계정 생성/삭제 등을 현장에서 바로 시뮬레이션할 수 있습니다.
*   특히, **운행 분석 RAG 에이전트 스트리밍 API** 호출 시, Server-Sent Events(SSE) 이벤트 연결을 맺어 RAG 추론 텍스트 결과물을 실시간 한 줄씩 누적하여 마치 터미널 콘솔처럼 시각적으로 렌더링해 줍니다.

## 4. E2E 통합 테스트 검증 시나리오

개발 서버가 구동 중인 상태에서 아래의 시나리오를 실행하여 구현 사항들을 실시간 검증할 수 있습니다.

### [시나리오 1: 기사 온보딩 및 맞춤 사주 브리핑 연동]
1. 프론트오피스([FO App](http://localhost:5173/onboarding)) 진입.
2. 생년월일 `YYYYMMDD` 입력 시 `-` 자동 완성, 출생시간 `HHMM` 입력 시 `:` 자동 완성을 확인하고 마스터 프로필 등록.
3. 선호 내비게이션(TMAP / 카카오네비)을 선택하여 저장.
4. 오늘의 루틴([Home](http://localhost:5173))에서 내가 등록한 생년월일을 기초로 만세력 사주 카드가 "최상", "상", "우수" 등 실시간 등급과 함께 정밀 로드되는지 확인.
5. 상단 헤더에 오늘 날짜와 한글 요일(예: `2026년 6월 20일 토요일`)이 테두리가 씌워진 골드 배지로 크고 세련되게 나오는지 검증.
6. 만세력 사주 브리핑 라벨("운세 브리핑"), 우회 경로 브리핑 라벨("아침 핵심 추천 코스"), 피드 통계("조회수 1.2만")의 가독성이 한눈에 들어올 만큼 뚜렷해졌는지 검증.
7. "경로 탐색 시작"을 눌러 설정한 내비 preference 스키마 딥링크가 동적으로 발현되는지 확인.

### [시나리오 2: G-PAN 레이더 실시간 대기 갱신 및 TTS 방송]
1. [G-PAN 레이더 페이지](http://localhost:5173/gpan) 진입 시 실시간 핫존(강남역, 김포공항) 목록 로드 확인.
2. API 서버가 비활성화되거나 오류를 반환할 때 상단에 `⚠️ LOCAL CACHE ACTIVE` 경고가 발생하며 로컬스토리지를 통해 끊김 없는 뷰가 가동되는지 확인.
3. `ON AIR` 재생을 누르면 부드럽게 조정된 여성 목소리로 핫존 예상 대기 시간과 교통 상황이 연이어 동적 낭독되는지 확인.

### [시나리오 3: 로드보더 OCR 인증 및 매출 리더보드 등극]
1. [로드보더 페이지](http://localhost:5173/plaza) 진입 후 "영수증 OCR 인증" 클릭.
2. 모바일 장비에서 파일 가져오기(갤러리/폴더) 또는 카메라 직접 촬영 다중 선택권이 정상 발현되는지 확인.
3. OCR 분석이 끝나고 ₩ 54,200 금액이 검출되면, 리더보드 랭킹 목록이 자동으로 실시간 갱신되어 나(기사 프로필의 홈택스 마스킹 정보)가 1위로 등극하는지 검증.

### [시나리오 4: BO 로그인 및 계정 등록 관리]
1. 백오피스([BO App](http://localhost:5174)) 진입 시 로그인 페이지에서 "시스템 로그인" 실행.
2. 사이드바 하단 "관리자 계정 관리" 클릭 후 임의의 관리자(예: 이메일 `manager2@unsu.com`, 역할 `Manager`)를 생성.
3. 새로 추가된 관리자 계정으로 로그아웃 후 다시 로그인을 테스트.

### [시나리오 5: BO 회원정보 관리, 권한별 마스킹 및 변경 이력 모달 가동]
1. `Super Admin` 계정(`admin@unsu-platform.com`)으로 로그인하여 **"기사 정보 관리"** 진입 후 기사 ID(`driver-101`)를 조회.
2. 주민번호(홈택스 ID)가 평문으로 정상 노출되며, 수정 에디터에서도 입력 수정이 가능한지 확인 후 생년월일이나 영업유형을 변경하여 저장.
3. 로그아웃 후 방금 새로 만든 `Manager` 권한 관리자 계정으로 로그인하여 동일하게 `driver-101` 기사 정보를 조회.
4. 주민번호(홈택스 ID)가 `driver-101` 데이터 요약 및 수정 에디터 폼 양쪽 모두 앞 3자리 제외 마스킹(`abc******`) 처리되는지 확인하고, 수정 폼의 홈택스 ID 입력 필드가 비활성화(`disabled`/`readOnly`) 처리되어 조작할 수 없는지 검증.
5. **"변경 이력 보기 (Audit History)"** 버튼을 클릭하여 방금 `Super Admin` 권한으로 수정한 변경 전후 필드 값 변화 상세 내용, 변경자 이메일, 변경 시각이 모달 내 타임라인 테이블에 온전히 출력되는지 E2E 최종 확인.
