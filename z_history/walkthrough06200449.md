# UNSU 플랫폼 실데이터베이스 및 REST API 통합 연동 완료 리포트

본 문서는 UNSU 플랫폼의 프론트엔드 모바일 클라이언트(FO)와 본사 백오피스(BO)가 로컬 모의 상태(`localStorage`)를 탈피하여, 실제 백엔드 API 서버(Express) 및 데이터베이스(PostgreSQL/Supabase)와 완전 연동(E2E Integration)된 개발 및 설계 결과물에 대한 요약입니다.

---

## 1. 백엔드 데이터베이스 및 API 아키텍처 연동 내역

안정적인 데이터 관리와 B2B 파트너 정산의 법적/기술적 안정성을 위해 아래의 통합 아키텍처를 가동했습니다.

### ① DB 클라이언트 세션 래퍼 (`api/src/utils/db.ts`)
*   **Dual Mode 지원**: 환경 변수 `DATABASE_URL`이 있으면 Supabase PostgreSQL 커넥션을 우선 사용하고, 없을 시 서버 중단 없이 테스트할 수 있도록 로컬 파일 Fallback 데이터베이스(`api/src/utils/local_db.json`)로 즉시 대체 작동합니다.
*   **보안 규격**: 기사의 개인 프로필 데이터 저장 시 주민번호/생년월일 및 홈택스 식별키를 암호화 보관하거나 BO 관제 대시보드상에서 강제 마스킹 처리하여 PII 노출 리스크를 원천 차단했습니다.

### ② 백엔드 API 서비스 엔드포인트 (`api/src/server.ts`)
기능 정의서에 맞춘 모든 CRUD 라우트를 구현하고 프론트엔드 연동을 마쳤습니다.
*   `POST /api/drivers/:id` / `GET /api/drivers/:id`: 기사 마스터 프로필 가입 및 조회 (생년월일 `-`, 시간 `:` 정규식 포맷팅 대응)
*   `GET /api/routine/:driverId`: 기사별 생년월일 사주에 따른 맞춤형 오늘의 행운 카드 및 추천 이동 경로
*   `GET /api/gpan/hotzones` / `POST /api/gpan/update`: 실시간 GPS 관제 핫존 조회 및 대기 시간 수동 갱신 연동
*   `GET /api/board/leaderboard` / `POST /api/board/ocr`: 실시간 영수증 OCR 인증 대조 및 오늘 하루 매출 리더보드 순위 자동 재연산
*   `POST /api/admin/login`: BO 관리자 전용 권한 검증 및 테스트 기본 계정 로그인
*   `GET/POST/DELETE /api/admin/accounts`: 신규 어드민 매니저/오디터 추가 등록 및 파쇄 관리

## 2. 프론트엔드(FO) API 마이그레이션 완료 내역

로컬 시뮬레이션 상태로 돌아가던 핵심 화면들을 REST API 연동 구조로 전면 수정했습니다.

*   **기사 온보딩 (`OnboardingPage.tsx`)**: 브라우저의 고유 `driverId`를 발행 및 바인딩하여, 가입 단계에서 `POST /api/drivers/:id`를 호출해 DB에 영구 레코드를 적재합니다. 가입 후 기사가 재방문하면 자동으로 프로필 정보를 DB로부터 가져옵니다.
*   **오늘의 루틴 (`GillogPage.tsx`)**: DB 기사 사주에 따른 행운의 Comment와 최적 우회로 추천 T-MAP 경로를 서버 통신(`GET /api/routine/:driverId`)으로 받아와 시각화하며, 프로필이 미등록된 기사의 경우 온보딩 페이지로의 리다이렉트 가이드를 띄웁니다.
*   **G-PAN 레이더 (`GPanRadarPage.tsx`)**:
    *   실시간 핫존 현황을 `GET /api/gpan/hotzones`를 통해 실시간 로드합니다.
    *   `업데이트` 리프레시 버튼 클릭 시, 각 핫존의 대기시간 무작위 차이 보정치(±3분)를 계산하여 `POST /api/gpan/update` API를 날려 DB 레코드를 갱신한 후, 최신 핫존 목록을 재조회합니다.
    *   **음성 편의성 최적화**: 너무 기계적인 TTS 낭독 톤을 개선하기 위해, 한국어 여성 음성엔진(Yuna, Seoyeon, Heami 등)을 우선 바인딩하고, 재생 속도(`rate = 0.86`)와 피치(`pitch = 0.98`)를 미세 조정하여 부드럽고 차분한 방송 안내를 선사합니다.
*   **로드보더 리더보드 (`RoadboarderPage.tsx`)**:
    *   오늘 하루 탑 보더 목록을 `GET /api/board/leaderboard`로 호출해 실시간 랭킹 순서대로 정렬 렌더링합니다.
    *   카메라 영수증 업로드 시, 모의 업로드 루프를 타는 대신 `POST /api/board/ocr` API를 통해 기사 이름과 영수증 데이터를 백엔드에 제출하고, 즉각 리더보드를 리프레시하도록 연동 완료했습니다.

## 3. 백오피스(BO) API 마이그레이션 완료 내역

사이드바 로그인 게이트 및 신규 어드민 관리 모듈을 백엔드 DB 상태와 E2E 결합시켰습니다.

*   **어드민 로그인 게이트 (`App.tsx` > LoginPage)**:
    *   `POST /api/admin/login`을 호출해 관리자 로그인을 수행합니다.
    *   초기 최고관리자 계정(`admin@unsu-platform.com` / `admin-secure-unsu`)이 인풋에 자동 바인딩되어 원클릭으로 쉽게 테스트할 수 있습니다.
*   **실시간 어드민 계정 관리 (`App.tsx` > Layout)**:
    *   사이드바 하단 "관리자 계정 관리"를 열면 `GET /api/admin/accounts`를 호출하여 가입된 관리자 리스트를 실시간으로 받아옵니다.
    *   신규 매니저/오디터 관리자 가입 시 `POST /api/admin/accounts`로 신규 계정을 등록하며, 삭제 처리 시 `DELETE /api/admin/accounts/:email`로 영구 파쇄 처리한 뒤 목록을 다시 불러옵니다.
    *   추가된 관리자 계정들은 모두 `admin-secure-unsu` 패스워드를 사용하여 로그인 페이지를 통해 정상적으로 대시보드에 접근할 수 있게 개발되었습니다.

## 4. E2E 통합 테스트 검증 시나리오

개발 서버가 실시간으로 작동 중인 상태에서 아래와 같이 시나리오 테스트를 진행할 수 있습니다.

### [시나리오 1: 기사 온보딩 및 맞춤 사주 브리핑 연동]
1. 프론트오피스([FO App](http://localhost:5173/onboarding)) 진입.
2. 생년월일 `YYYYMMDD` 입력 시 `-` 자동 완성, 출생시간 `HHMM` 입력 시 `:` 자동 완성을 확인하고 마스터 프로필 등록.
3. 오늘의 루틴([Home](http://localhost:5173))에서 내가 등록한 생년월일을 기초로 만세력 사주 카드가 "최상", "상", "우수" 등 실시간 등급과 함께 정밀 로드되는지 확인.

### [시나리오 2: G-PAN 레이더 실시간 대기 갱신 및 TTS 방송]
1. [G-PAN 레이더 페이지](http://localhost:5173/gpan) 진입 시 실시간 핫존(강남역, 김포공항) 목록 로드 확인.
2. `업데이트` 버튼을 누르면 백엔드에 핫존 분량이 갱신(`POST /api/gpan/update`)된 후 리로드되는지 확인.
3. `ON AIR` 재생을 누르면 부드럽게 조정된 여성 목소리로 핫존 예상 대기 시간과 교통 상황이 연이어 동적 낭독되는지 확인.

### [시나리오 3: 로드보더 OCR 인증 및 매출 리더보드 등극]
1. [로드보더 페이지](http://localhost:5173/plaza) 진입 후 "영수증 OCR 인증" 클릭 및 이미지 업로드.
2. OCR 분석이 끝나고 ₩ 54,200 금액이 검출되면, 리더보드 랭킹 목록이 자동으로 실시간 갱신되어 나(기사 프로필의 홈택스 마스킹 정보)가 1위로 등극하는지 검증.

### [시나리오 4: BO 로그인 및 계정 등록 관리]
1. 백오피스([BO App](http://localhost:5174)) 진입 시 로그인 페이지에서 "시스템 로그인" 실행.
2. 사이드바 하단 "관리자 계정 관리" 클릭 후 `manager2@unsu.com` 등 임의의 관리자를 생성.
3. 추가된 계정이 리스트에 마킹되는지 확인하고, 로그아웃 후 새로 만든 이메일 계정으로도 로그인이 승인되는지 E2E 테스트.
