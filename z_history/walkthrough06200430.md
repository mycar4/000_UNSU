# UNSU 플랫폼 프론트엔드(FO) 및 백오피스(BO) 개발 요약

본 문서에는 엔터프라이즈 사양에 따라 개발된 프론트엔드 모바일 클라이언트(FO)의 튜닝 내역 및 새로 구축 완료된 백오피스 관리 시스템(BO)의 4대 관리 화면의 아키텍처와 검증 시나리오가 담겨 있습니다.

---

## 1. 프론트엔드(FO) 피드백 튜닝 완료 내역
- [x] **ON DUTY 상태 동기화**: 상단 바의 `ON DUTY` 뱃지가 G-PAN의 라디오 `ON AIR` 재생 상태와 전역 컨텍스트로 실시간 연동됩니다.
- [x] **온보딩 입력값 자동 포맷팅 마스킹**: 생년월일(`YYYY-MM-DD`) 및 출생 시간(`HH:MM`) 입력 시 숫자만 적으면 포맷팅 기호가 자동으로 동적 삽입됩니다.
- [x] **정보 수정 모드 및 초기화(로그아웃)**: 온보딩된 프로필이 존재할 시 "정보 변경 모드"로 작동하고 기존 데이터가 양식에 바인딩되며, 하단에 기사 테스트 초기화를 위한 로그아웃 버튼을 내장했습니다.
- [x] **가독성 날짜 노출**: 오늘의 일진 행운 카드 렌더링 시 `"01-01"` 형식을 `"01월 01일 생"` 포맷으로 변경해 부드러운 정서를 표현했습니다.
- [x] **TTS 음질 향상**: 한국어 여성 음성을 우선 감지(Yuna, Heami, Google 등)하고 배속(`0.90`) 및 피치(`1.05`)를 튜닝해 한결 부드럽고 맑은 한국어 해설 낭독을 보장합니다.
- [x] **G-PAN 실시간 핫존 수동 리프레시**: `업데이트` 버튼 구축 및 1초의 로더 연출 뒤 최근 갱신 완료 시각(`최근 업데이트 HH:MM:SS`)을 마킹하며 대기열 분 단위를 무작위로 변경합니다.
- [x] **영수증 OCR 카메라 촬영 연동**: 파일 input에 `capture="environment"` 속성을 믹스하여 실제 모바일 카메라 기기 렌더링을 대응했습니다.

## 2. 백오피스(BO) 4대 관리자 화면 구축 완료 내역
본사 어드민의 독립적이고 안전한 비즈니스 관제를 위해 아래 4대 관리 화면을 신규 개발하고 라우팅을 완전 결합했습니다.

### ① 세무 자율비행 정산 관제 ([TaxAutopilotMonitor](file:///d:/000_UNSU/bo/src/pages/TaxAutopilotMonitor.tsx))
- **PII Zero-Storage 규격**: 기사 개인정보 유출을 방지하기 위해 홈택스 ID와 기사명을 전부 마스킹(`kims***`, `김*수`)하여 화면상에 렌더링합니다.
- **배치 상태 모니터링**: 핀테크(CODEF/쿠콘) 연동 상태 및 정산 이력의 상태(성공, 보류, 오류) 비율을 요약 출력합니다.
- **수동 재발송 트리거**: `임시 보류건 수동 재발송` 버튼을 누르면, confirm 승인을 거친 후 1.2초간 홈택스 API 재전송 모션이 구동되고, 대기 및 오류 건들을 성공(`SUCCESS`)으로 즉시 동기화합니다.

### ② B2B 제휴사 정산 관리 ([PartnerSettlement](file:///d:/000_UNSU/bo/src/pages/PartnerSettlement.tsx))
- **제휴 수익 계산**: 타이어 몰, 정비 인프라, 제휴 세무사 등 B2B 제휴사의 실시간 매출 데이터에 연동된 플랫폼 수수료율(5%~12%)을 동적 연산하여 플랫폼 수수료 수익 및 대기 송금액을 연출합니다.
- **파괴적 송금 액션 안전 가드**: `정산 실행` 클릭 시, 제휴처 명칭과 지급금액을 재확인시키는 2차 경고 컴포넌트 프롬프트가 작동하여 오송금을 원천 예방합니다.

### ③ 커뮤니티 및 영수증 신뢰 모니터링 ([PlazaModeration](file:///d:/000_UNSU/bo/src/pages/PlazaModeration.tsx))
- **영수증 OCR 오차율(Discrepancy) 자동 판독**: 기사 업로드 영수증 OCR 금액과 금융망 카드 결사 실데이터를 정밀 대조합니다. 오차가 5%를 초과할 경우(예: OCR 85,000원 vs 실제 25,000원) **"오차 240% 검출 (경고)"** 뱃지를 빨간색으로 점멸시킵니다.
- **어뷰징 제재**: 허위 인증 어뷰징 기사 제재 버튼 클릭 시, 2차 confirm 확인 후 리더보드 자격을 박탈하고 정지 처리를 이행합니다.
- **기사 광장 블라인드**: 비속어 및 욕설로 신고된 자유게시판 글을 일괄 모니터링하고 즉시 숨김(`BLINDED`) 처리 또는 기각 처리합니다.

### ④ G-PAN 실시간 관제 및 AI 피드백 튜닝 ([GPanStatusDashboard](file:///d:/000_UNSU/bo/src/pages/GPanStatusDashboard.tsx))
- **API 생존율 통계**: 기사 및 서울시 TOPIS 소통 API 게이트웨이의 지연 시간(latency) 및 헬스 스코어를 수치화하여 모니터링합니다.
- **LangSmith 피드백 플라이휠 연계**: 주행 기사들이 핫존 오더 수주에 실패할 시 수집되는 **"허탕 통계"** GPS 위치 좌표 데이터셋 수집 현황을 실시간 인덱싱합니다.
- **가중치 파인튜닝 동기화**: `가중치 미세 조정 동기화`를 클릭하면, confirm 후 1.5초간 AI 모델 가중치 미세 조정 학습이 가동되고, 에이전트 프롬프트 버전(예: `v1.4.1` ➔ `v1.5`)이 실시간 핫패치되어 배포 완료됩니다.

## 3. 라우팅 및 사이드바 결합 ([App](file:///d:/000_UNSU/bo/src/App.tsx))
- 백오피스 사이드바의 링커들을 `정산 관제`, `제휴사 정산`, `커뮤니티 및 OCR`, `G-PAN 대시보드`로 변경하고, `/`, `/partner`, `/plaza`, `/gpan` 각각의 경로로 독립된 4개 어드민 화면 라우팅을 완전 가동했습니다.
- **[NEW] 백오피스 로그인 게이트**: BO 진입 시 관리자 전용 로그인 페이지를 노출합니다. 테스트 편의성을 위해 기본 슈퍼 관리자 계정(`admin@unsu-platform.com` / `admin-secure-unsu`)을 기본값으로 입력해 두어 원클릭 진입이 가능하도록 지원합니다.
- **[NEW] 관리자 계정 추가 등록 관리**: 사이드바 하단의 "관리자 계정 관리" 버튼을 클릭 시 관리자 권한을 가진 신규 이메일 계정을 실시간으로 등록 및 삭제할 수 있는 계정 제어 모듈을 구축했습니다.

### 👉 BO 테스트 가이드
로컬 서버 관리자 브라우저([Vite BO App](http://localhost:5174) 혹은 포트별로 구동되는 Admin Page)로 진입하셔서 아래 시나리오를 테스트해 보세요:
1. **로그인 게이트**: BO 페이지 접속 시 로그인 카드가 표시되는지 확인하고, 디폴트로 채워진 로그인 정보 그대로 "시스템 로그인"을 누릅니다.
2. **관리자 추가 등록**: 사이드바 왼쪽 하단 "관리자 계정 관리"를 눌러 신규 관리자 이름과 이메일을 입력한 뒤 "계정 추가"를 수행하여 목록에 반영되고 로그아웃 후 해당 메일로도 신규 로그인이 가능한지 교차 검증합니다.
3. **정산 관제**: `임시 보류건 수동 재발송` 버튼을 누르면 API 전송 애니메이션 후 오류/대기 기사들의 상태가 실시간 성공 처리되며 CODEF 에러율이 0%로 떨어지는 통계 역학을 검증합니다.
4. **제휴사 정산**: 지급 대기 파트너사의 `정산 실행` 단추를 누르고 2차 송금 알림창을 승인하여 정산 완료로 상태가 갱신되는지 확인합니다.
5. **커뮤니티 및 OCR**: 청라국제도시 기사의 영수증 오차 경고 배너를 확인하고 `어뷰징 제재`를 가동해 자격 박탈 상태로 변이하는지 검증합니다. 아래 기사 광장 신고 글들의 블라인드 처리 토글을 가동합니다.
6. **G-PAN 대시보드**: 누적된 허탕 수집 인덱스를 모니터링하고 `가중치 미세 조정 동기화`를 실행하여 1.5초의 모델 학습 모션 후 프롬프트 버전이 동적으로 상승 갱신되는 것을 검증합니다.
