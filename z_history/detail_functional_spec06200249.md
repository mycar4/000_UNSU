# [Spec] UNSU 플랫폼 4대 핵심 메뉴 상세 기능 정의서 (v1.0)

본 문서는 UNSU 프론트오피스(FO)의 4대 핵심 메뉴에 대한 구체적인 비즈니스 로직, 데이터 입출력 명세, 에이전트(LangGraph) 연계 흐름, 그리고 결함 격리(Fallback) 정책을 규정하는 싱글 소스 오브 트루스(SSOT) 기능 명세서입니다.

---

## 1. MENU 01: 오늘의 루틴 (GILLOG)

### 1-1. 기물 사주 기반 행운 카드 자동 생성
*   **비즈니스 목적**: 기사의 정서적 락인(Lock-in) 및 일간 인게이지먼트 유도.
*   **작동 흐름**:
    1.  기사가 출근(앱 접속) 시, 기사 프로필 데이터(생년월일시, 성별)와 오늘의 일진(육십간지) 데이터를 수집합니다.
    2.  LangChain LLM 체인(에이전트)에 사주 텍스트 맥락을 입력하여 오늘의 재물운 등급(최상/상/중/하) 및 일일 행동 지침 코멘트를 생성합니다.
*   **AI 프롬프트 파라미터 구조**:
    ```json
    {
      "driver_birth_date": "1968-04-12",
      "driver_birth_time": "14:30",
      "daily_solunar_info": "병자일 (丙子日)",
      "target_location": "서울 강남구"
    }
    ```
*   **예외 처리 (Fallback)**: 사주 파싱 API 또는 LLM 타임아웃(5초) 초과 시, 데이터베이스에 기저장된 기사 생년월일 기준의 12간지 기본 행운 코멘트 템플릿("오늘은 서쪽 방향의 운행이 순탄합니다. 안전운전 하세요.")을 즉시 대체 송출합니다.

### 1-2. 아침 핵심 추천 코스 및 TMap 인텐트 연동
*   **비즈니스 목적**: 출근길 첫 장거리 콜 매칭 확률 극대화.
*   **작동 흐름**:
    1.  매일 오전 06:30 기상청 날씨, 서울시 토폴로지 도로 교통 정보(TOPIS API)를 비동기로 스크래핑합니다.
    2.  당일 아침의 메이저 혼잡 유발 이벤트(예: 지하철 파업, 올림픽대로 공사 등)를 반영하여 최종 추천 도착 랜드마크(예: 김포공항)와 우회 경로를 산출합니다.
    3.  "추천 경로 티맵 전송" 버튼 클릭 시, 기사의 스마트폰에 설치된 TMap 앱으로 목적지 좌표를 다이렉트 바인딩(Intent Link)합니다.
*   **보안 Baseline (WHATWG)**: TMap 스키마 호출 URL 생성 시, 임의의 스크립트 주입 방지를 위해 화이트리스트 검사(`tmap://route?rGoName=...&rGoX=...&rGoY=...`)를 수행하며, 외부 웹 링크 이동 시 `target="_blank" rel="noopener noreferrer"`를 강제 바인딩합니다.

## 2. MENU 02: G-PAN 레이더 (실시간 관제)

### 2-1. Zero-Touch 오디오 관제 엔진 및 미디어 플레이어
*   **비즈니스 목적**: 주행 중 수동 조작을 배제한 전방 주시형 안전 TTS 방송 송출.
*   **플레이어 상태 머신 (State Machine)**:
    ```
    [IDLE] ➔ Play Click ➔ [LOADING] ➔ Streaming Start ➔ [PLAYING]
      ▲                      │                               │
      │                  Timeout/Network Error               │
      └──────────────────────┴──────── [ERROR] ◄─────────────┘
    ```
*   **오디오 변환(TTS) 파이프라인**:
    *   스크래핑된 교통 맥락 텍스트(예: "현재 올림픽대로 가양대교 부근 낙하물 사고로 3차선 통제 중...")를 OpenAI Audio Speech 또는 Google Cloud TTS API로 전송하여 오디오 버퍼 스트림을 실시간 생성합니다.
*   **예외 처리 (Fallback)**: 
    *   TTS 변환 실패 또는 모바일 셀룰러 음영 구역 진입으로 인한 오디오 스트리밍 버퍼링 지연(3초 이상) 시, 자동으로 오디오 엔진을 중지하고 **텍스트 전광판 알림 UI**와 **기본 경고 비프음(Beep Sound)**으로 기사의 시각적 인지를 유도합니다.

### 2-2. 실시간 핫존 및 대기열 동기화
*   **데이터 갱신 주기**: 핫존 정보는 서버 부하 및 배터리 절약을 위해 **5분 주기**로 폴링(Polling)하여 UI를 동적 리렌더링합니다.
*   **GPS 단절 시 예외 처리**: 터널 또는 지하 주차장 등 GPS 음영 구역에서 핫존 검색 요청 시, **마지막으로 수집된 기사의 유효 GPS 좌표(Last Known Location)**를 기준으로 핫존 데이터를 서빙하고 화면에 "GPS 수신 불안정 - 최종 위치 기준" 경고 뱃지를 출력합니다.

## 3. MENU 03: 로드보더 (리더보드 & 커뮤니티)

### 3-1. 당일 최고 매출 리더보드 정산 프로세스
*   **정산 마감 타이밍**: 매일 아침 **AM 09:00**에 전날(00:00 ~ 23:59) 매출 정산을 자동으로 마감하고 상위 1, 2, 3위 보더를 마스터 리더보드에 갱신 및 박제합니다.
*   **매출 신뢰성 검증 (어뷰징 차단)**: 
    *   기사가 매출액을 수동으로 속여서 입력하는 것을 차단하기 위해, 기사 카드단말기 승인 내역 API(여신금융협회 스크래핑) 또는 디지털 운행기록계(DTG) 상의 실운행 거리 대비 매출 비율(1km당 평균 1,500원 ~ 3,000원 범위)의 2차 검증 필터를 통과해야만 리더보드 등극 자격을 부여합니다.

### 3-2. 기사 광장 타임라인 및 보안 필터링
*   **중복 좋아요 방지**: `PostLikes` 테이블에서 `(post_id, driver_id)` 복합 기본키(Composite Primary Key)를 설정하여 기사 1인당 1게시글 1좋아요 처리를 데이터베이스 레벨에서 원천 강제합니다.
*   **글로벌 보안 Baseline (DOMPurify & Zod)**:
    *   커뮤니티 글 입력 폼 전송 시 `Zod` 스키마 밸리데이션(글자수 제한 2,000자, 특수 문자 및 스크립트 태그 유입 원천 금지)을 강행합니다.
    *   기사가 작성한 텍스트를 화면에 렌더링할 때, React 컴포넌트 내부에서 임의의 태그 해석을 배제하며, 추후 HTML 마크업 렌더링이 필요한 부분은 사전에 `DOMPurify.sanitize()` 파이프라인을 100% 통과하도록 설계합니다.

## 4. MENU 04: 오토파일럿 (경영/정산)

### 4-1. 1초 간편 홈택스 정산 동기화
*   **CODEF/쿠콘 API 연동 시나리오**:
    1.  기사가 "1초 만에 자동 정산하기" 버튼 클릭 시, 백엔드에서는 공인인증서/간편인증 세션을 가동하여 홈택스 내 분기별 매입 세금계산서 및 신용카드 매입 내역 데이터를 스크래핑합니다.
    2.  수집된 데이터에서 '택시 차량 구입비', '충전소/주유소 이용액', '타이어 및 정비 공임' 등 **택시 사업자 매입세액 공제 대상 항목**만 추출하여 예상 환급액을 도출합니다.
*   **홈택스 점검 시간 대응**:
    *   국세청 홈택스 점검 시간(매일 00:00 ~ 09:00 등) 내 정산 요청이 올 경우, 프론트엔드에서는 **"국세청 홈택스 점검 시간입니다. 예약 정산으로 전환됩니다."** 안내 메시지를 출력합니다.
    *   백엔드에서는 해당 기사의 정산 요청 상태를 `정산 대기 (Pending)`로 적재하고, 오전 09:00 점검 해제 직후 배치 워커(Batch Worker)를 통해 자동 정산을 수행하고 푸시 알림을 발송합니다.

### 4-2. 예상 환급액 모의 산식 및 제휴 혜택 인터랙션
*   **환급 산식**: `예상 환급액 = (적격 매입 세금계산서 상의 부가세 총합) + (사업용 신용카드 매입 부가세 총합) - (매출 세액)`
*   **제휴 혜택 링크 안전 격리**:
    *   제휴 타이어몰 및 정비소 링크 아웃바운드 포워딩 시, `WHATWG URL` 모듈을 통한 `https:` 화이트리스트 검사를 수행하고 `target="_blank" rel="noopener noreferrer"` 속성을 강제하여 기사 단말기의 세션 제어권 탈취를 완벽히 격리 예방합니다.
