# [Walkthrough] FO 레이아웃 고정 및 메인바디 폰트 크기 불일치 해결 완료

기존 디자인 프리미엄 개편 이후, PC 데스크톱과 모바일 실기기 각각에서 보았을 때 발생하던 레이아웃 어긋남 및 메인바디 폰트의 크기가 "들쑥날쑥"해지는 문제를 완전히 수정하였습니다.

> [!TIP]
> 변경된 내역은 로컬 개발 환경(http://localhost:5173)을 열어 브라우저를 확인하시거나 개발자 도구(F12)로 가로 너비를 조절해 보시면서 확인하실 수 있습니다!

## 1. 주요 수정 및 고도화 내역 (Changes Made)

### 1-1. 📱 PC 브라우저 내 모바일 프레임 정렬 보정
* **수정 내용**: 
  - `AppLayout.tsx`, `TopAppBar.tsx`, `BottomNavBar.tsx` 등 모든 Fixed UI 컴포넌트에 `left-1/2 -translate-x-1/2` 클래스를 추가하여 PC 가로 뷰포트에서도 `max-w-md` 모바일 정중앙 영역 내에 완벽하게 일치하고 정렬되도록 조정했습니다.
  - 챗봇 팝업 모달이 팝업될 때 데스크톱용 반응형 클래스(`md:`)를 걷어내고, 항상 모바일 프레임 하단에 알맞게 정렬되도록 보정했습니다.

### 1-2. 🔤 메인바디 폰트 크기 "들쑥날쑥" 현상 해결
* **수정 내용**: 
  - `GillogPage.tsx`, `HomePage.tsx`, `AutopilotPage.tsx`, `DarksidePage.tsx`, `OnboardingPage.tsx`에서 모든 `sm:`, `md:` 미디어 쿼리 접두사를 제거했습니다.
  - PC 브라우저와 모바일 기기 모두에서 가로 배치와 세로 스크롤링이 모바일 1열 모드로 완벽히 동일하게 노출되도록 강제했습니다.
  - `DESIGN.md`에 명시된 시니어 가독성 토큰인 `text-body-lg` (20px, line-height 1.8)를 주요 본문 영역에 일관되게 입혀서 텍스트 크기가 갑자기 작아지거나 들쑥날쑥해지는 문제를 바로잡았습니다.

### 1-3. 🎨 타이틀 폰트 비대화 및 로고 찌그러짐 현상 최종 수정
* **대형 타이틀 폰트 스케일 고정**: 
  - `.hero-head` 스타일에 적용되어 있던 뷰포트 너비 비례 폰트 크기(`clamp(..., 9vw, ...)`)로 인해, PC 브라우저(F12 에뮬레이션을 켜지 않은 원본 데스크톱 화면)로 볼 때 글씨가 88px(5.5rem) 이상으로 비대해져 타이틀이 쪼개지던 현상을 발견했습니다.
  - 이를 방지하기 위해 `index.css`를 수정하여 `.hero-head`가 뷰포트 너비의 영향을 받지 않고 모바일 프레임에 적합한 절대 크기(`2.25rem` = 36px) 및 `line-height: 1.1`로 유지되도록 조율하였습니다.
* **상단 로고의 의도적인 세로 2줄화 및 원래 크기 복원**:
  - 줄바꿈이 없던 로고 형태 대신, 기사님의 선호에 따라 **"운수대통"이 세로 2줄로 정돈되게 흐르도록 디자인을 개편**했습니다.
  - 2줄 로고 상태에서도 시각적 존재감과 가독성을 확실하게 보장하기 위해, **처음 다운로드받았을 때의 원래 큼직한 로고 크기인 `text-xl` (20px)** 서체 크기를 고스란히 복원하여 적용했습니다.
  - "운수 / 대통"을 굵고 시원시원한 세로 서체로 묶고 그 옆에 얇은 수직 분할선(Border)과 함께 "AI PLATFORM"을 조화롭게 가로 배치하여 Warm Editorial 감성을 살렸습니다.
* **상단 테마 버튼 간소화**:
  - 기존 3개 스위치 패널(낮/노을/밤)을 **단일 순환 토글 버튼(클릭 시 테마 순차 변경)**으로 슬림화하여 로고가 위치할 가로 공간을 대폭 확보했습니다.
* **로드보더 페이지 상단 버튼 및 타이틀 배치 개선**:
  - 큰 제목인 "로드보더" 바로 옆에 "핫존 허탕 피드백" 버튼이 오면서 텍스트가 엉키던 레이아웃을, 소제목 우측으로 버튼 위치를 세로 분리하여 한층 정돈된 미려한 레이아웃을 연출했습니다.

### 1-4. 👆 본문 영역 좌우 스크롤(Swipe) 메뉴 이동 기능 탑재
* **수정 내용**: 
  - 모바일 사용자 경험(UX) 극대화를 위해 화면을 좌/우로 휙 쓸었을 때 하단 탭 메뉴(`루틴 ↔ G-PAN ↔ 로드보더 ↔ 정산`)로 순차 이동하는 터치 스와이프 이벤트를 `AppLayout.tsx` 컨테이너에 직접 구현했습니다.
  - 세로 스크롤 시 오작동하는 일이 없도록 **가로 움직임(x)이 60px 이상 확실하게 발생하고, 세로 움직임(y)은 40px 미만인 정교한 드래그 제스처만 감지**하여 이동하도록 세심하게 튜닝했습니다.

### 1-5. 📐 레이아웃 패딩 시스템 단일화 (px-5)
* **수정 내용**:
  - 상단바(`TopAppBar` - 24px 패딩), 본문 페이지(20px 패딩), 하단바(`BottomNavBar` - 16px 패딩)로 제각각 나뉘어 있어 페이지 이동이나 스크롤 시 정렬선이 안 맞고 미세하게 덜컹거리던 원인을 진단했습니다.
  - 상단바, 하단바, 챗봇 모달 백드롭 및 본문 페이지의 좌우 시작 여백을 **전부 `px-5` (20px)로 칼같이 통일**했습니다.
  - 추가적으로 `/board` (로드보더) 페이지 내부 카드 요소들의 제각각이던 여백(`p-4`, `p-6` 등) 또한 **모두 `p-5`로 통일**하여, 카드의 가로 끝 정렬 라인이 본문 시작선과 한 오차도 없이 일직선으로 딱 맞아떨어집니다.

### 1-6. 🌫️ 개별 페이지의 중복 격자 무늬 (.grid-lines) 배경 제거
* **수정 내용**:
  - 공통 레이아웃(`AppLayout.tsx`)에 깔려 있는 기본 격자 배경 위에, 개별 페이지들(`HomePage`, `AutopilotPage`, `RoadboarderPage`, `GPanRadarPage`)이 제각각 다른 투명도로 격자선을 한 번 더 중복해서 렌더링하고 있었습니다.
  - 이로 인해 메뉴 이동이나 슬라이드 시 배경 격자가 어긋나거나 선명도가 순간적으로 튀던 버그를 완전히 해결하기 위해 **모든 개별 페이지 내 중복 격자선 코드를 삭제**하고 레이아웃을 단일화했습니다.

### 1-7. 🔀 스크롤바 유무에 따른 좌우 화면 흔들림(덜컹거림) 원천 해제
* **수정 내용**:
  - 콘텐츠 길이가 긴 `/board` (로드보더)로 넘어갈 때만 우측 브라우저 세로 스크롤바가 생겨나 뷰포트 너비가 좁아지고, 이로 인해 `left-1/2 -translate-x-1/2`로 고정된 프레임들이 좌우로 미세하게 `덜컹` 흔들리는 현상을 발견했습니다.
  - 이를 원천 예방하기 위해 [index.css](file:///c:/000_UNSU/fo/src/index.css)의 `html` 요소에 최신 웹 표준 스크롤 공간 예약 속성인 **`scrollbar-gutter: stable`**과 크로스 브라우징을 위한 **`overflow-y: scroll`**을 강제 부여했습니다.
  - 이제 스크롤바가 나타나든 사라지든 항상 동일한 가로 영역 뷰포트를 확보하여, 메뉴 전환 시 어떠한 좌우 흔들림(덜컹거림)도 발생하지 않습니다.

### 2-2. 수동 레이아웃 검증
* PC 일반 브라우저 화면(F12 비활성화 상태)에서 `localhost:5173/board` 및 `/autopilot`에 접속했을 때 대형 폰트가 깨지지 않고 깔끔한 단일 칼럼 구조로 렌더링됨을 확인했습니다.
