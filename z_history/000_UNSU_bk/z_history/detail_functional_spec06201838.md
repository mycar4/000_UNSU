# [Spec] UNSU 플랫폼 4대 핵심 메뉴 상세 기능 정의서 (v2.0)

본 문서는 UNSU 프론트오피스(FO)의 4대 핵심 메뉴 및 마스터 온보딩에 대한 구체적인 비즈니스 로직, 데이터 입출력 명세, 에이전트(LangGraph) 연계 흐름, 그리고 결함 격리(Fallback) 정책을 규정하는 싱글 소스 오브 트루스(SSOT) 기능 명세서입니다.

---

## 1. MENU 01: 오늘의 루틴 (GILLOG)

### 1-1. 정적 만세력 알고리즘 + Gemini 맥락 결합 (사주 API 비용 절감)
*   **비즈니스 목적**: 외부 사주 API의 호출당 비용 부담을 완전히 제거하고, 기계적이고 과학적인 일일 행운 동선 내러티브 생성.
*   **작동 흐름**:
    1.  **정적 수식 계산**: 외주 API 연동을 배제하고 백엔드 엔진(`api/`) 내부에 한국천문연구원 기준 정적 만세력 계산 모듈을 탑재하여 기사의 생년월일시 정보로부터 육십간지(일진) 코드를 도출합니다.
    2.  **맥락 결합 (Context Merging)**: 산출된 간지 코드 텍스트(예: "을축일")와 기상 정보, 트래픽 실시간 변수를 Gemini API 프롬프트에 결합합니다.
    3.  **내러티브 렌더링**: 추가 호출 비용 없이, 고정된 LLM 토큰 예산 내에서 개인 맞춤형 "데이터 과학 기반 행운 동선" 가이드 텍스트를 빌드합니다.
*   **예외 처리 (Fallback)**: 네트워크 단절 또는 LLM 런타임 예외 발생 시, 로컬 Vector DB에 미리 적재된 60갑자별 '안전 운전 확언 템플릿'으로 즉시 스위칭하여 무중단 서비스를 제공합니다.

### 1-2. 아침 핵심 추천 코스 및 TMap 인텐트 연동
*   **작동 흐름**: 기상청 날씨 및 서울시 TOPIS 실시간 트래픽 데이터를 분석하여 당일 아침의 혼잡 돌발 이벤트를 우회하는 최종 도착 랜드마크를 기사 화면에 브리핑하고, 클릭 시 모바일 TMap 앱 내비게이션 좌표로 안전 바인딩(`tmap://`)합니다.

## 2. MENU 02: G-PAN 레이더 (실시간 관제)

### 2-1. 실시간 관제 데이터 수집 매트릭스
*   수집 채널별 동기화 주기 및 서버 캐싱 범위 규격:

| 데이터 분류 | 갱신 주기 (Cycle) | 데이터 관리 및 캐싱 전략 |
| --- | --- | --- |
| **초고동적 데이터**<br>(기상청 날씨, 서울 TOPIS 돌발 사고) | 5분 주기 Polling | Supabase 임시 캐시 테이블 적재. 기사의 현재 GPS 반경 5km 이내의 변동 사양만 필터링 후 밀어넣기. |
| **정기/사전공지 데이터**<br>(KTX/지하철 지연, 공항 연착, 축제/공연) | 12시간 주기 Batch | 서울 오픈데이터 광장 및 한국공항공사 API를 일일 새벽 스케줄러로 긁어모아 로컬 DB에 영속화. |

### 2-2. 클라이언트 Native TTS 합성 (오디오 결함 격리 및 트래픽 절감)
*   **아키텍처 선회**: 서버에서 대용량 mp3/wav 오디오 파일을 생성하여 전송하는 고비용 스트리밍 방식을 폐기합니다.
*   **작동 메커니즘**: 백엔드는 정제된 문자열 데이터(Text Payload)만 안전하게 모바일 브라우저로 전송합니다. 프론트엔드는 HTML5 브라우저 내장 **Web Speech API (SpeechSynthesis)**를 호출하여 로컬 하드웨어 리소스로 음성을 자체 합성 및 낭독합니다.
*   **효과**: 통신 음영 지대(터널, 지하 등)에서도 오디오가 중간에 뚝뚝 끊기는 결함을 완벽히 원천 차단하며, 서버의 스트리밍 트래픽 비용을 90% 이상 획기적으로 절감합니다.

## 3. MENU 03: 로드보더 (리더보드 & 커뮤니티)

### 3-1. 매출 검증 현실성 보완: "영수증 OCR 스캔 + 사후 대조"
*   **대체 검증 프로세스**: 민간 연동이 까다로운 CREFIA 및 DTG 실시간 강제 연동 대신, 기사들의 입력 진입 장벽을 낮추고 오염 데이터를 방지하는 **"간편 영수증 OCR 스캔"**을 1차 인증 수단으로 설정합니다.
*   **동작 파이프라인**:
    1.  기사가 모바일 카메라로 당일 카드단말기 결산 영수증을 스캔합니다.
    2.  프론트엔드에서 이미지 리사이징 후 백엔드 OCR 엔진 노드로 압축 전송합니다.
    3.  승인 금액 및 날짜 자동 인식 후 리더보드에 즉시 임시 등록 처리합니다.
    4.  퇴근 시점에 오토파일럿이 쿠콘 API 등으로 수집한 공인 정산 데이터와 비교 검증을 거쳐 사후 어뷰징 기사를 랭킹에서 박탈 및 패널티를 부여합니다.

### 3-2. LangSmith 기반 사용자 피드백 플라이휠 (데이터 개선 순환)
*   기사가 핫존 추천에 갔으나 실제 콜 수주에 실패했을 때 누르는 **"허탕 아이콘(Interaction Tool)"** 피드백 데이터를 수집합니다.
*   이 데이터는 `LangSmith Dataset`에 트레이싱셋으로 실시간 누적 적재되며, 주기적으로 Gemini 에이전트 프롬프트 가중치를 파인튜닝(Fine-tuning)하여 핫존 예측 정확도를 끌어올리는 순환 루프(Feedback Flywheel)를 이룹니다.

## 4. MENU 04: 오토파일럿 (경영/정산)

### 4-1. PII 개인정보 보호 정책: Zero-Storage Policy (임시 세션 처리)
*   **데이터 보호 가드레일**: 홈택스 간편인증 스크래핑 과정에서 발생할 수 있는 기사의 고유 식별 정보(주민등록번호 등) 및 일회성 세션 토큰은 **절대 데이터베이스(Supabase)에 영구 저장하지 않습니다.**
*   **메모리 격리**: 스크래핑 요청 런타임에 한해서만 메모리 상에 일시 적재하여 금융 브로커(CODEF 등)로 전송한 후, 데이터 수집 완료 즉시 가비지 컬렉터(GC)에 의해 메모리 공간을 완전히 영(Zero)으로 소멸시킵니다.

### 4-2. 정밀화된 개인택시 간이과세자 부가세 환급 산식
*   일반과세자와 다른 개인택시 간이과세자의 특수성을 반영한 부가세 환급 모의 계산식을 공식화합니다.
*   **산식**:
    $$\text{예상 환급액} = \sum (\text{적격 매입 부가세액}) \times 0.5\ (\text{간이과세자 신용카드 등 공제 가중치}) - (\text{매출 세액} \times \text{개인택시 업종별 부가가치율})$$
*   **상시 노출의 비즈니스 가치**: 환급액을 월간/일간 가산 방식으로 대시보드 전면에 상시 노출하여 기사님이 비용 절감 지표를 매일 직관적으로 모니터링할 수 있도록 하여 서비스 리텐션(Retention)을 향상합니다.

## 5. FO-00-ONBOARDING: 기사 마스터 프로필 온보딩 시스템

### 5-1. UI/UX 디자인 가이드라인
*   **미니멀리즘 인터페이스**: 복잡한 입력 양식으로 인한 시니어 기사 이탈 방지를 위해 1화면 1입력을 제공하는 단계형(Step-by-Step) 전개.
*   **Stitch 화이트 테마**: Cream 기반 배경(`bg-background`), 넓은 인풋 패딩, 가독성 높은 헤드라인 서체 조합 및 부드러운 스케일 트랙(`.tap`) 적용.
