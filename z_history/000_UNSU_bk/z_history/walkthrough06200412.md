# UNSU 프론트엔드 실시간 Mock API 및 인터랙티브 피쳐 개발 및 피드백 튜닝 요약

프론트엔드 우선 개발 정책에 의거하여, 단순한 시각 목업 수준을 넘어 실제 기사가 조작할 수 있는 **동적 상태 전이(State Transition) 및 브라우저 네이티브 하드웨어 API 연동**을 구현하고, 사용자 피드백을 기반으로 한 고도화 튜닝 작업을 마쳤습니다.

---

## 1. 구현된 핵심 인터랙티브 기능 및 피드백 튜닝 사항

### ① 기사 온보딩 마스터 프로필 연동 ([OnboardingPage](file:///d:/000_UNSU/fo/src/pages/OnboardingPage.tsx))
*   **입력 자동 포맷팅 마스킹**: 생년월일 입력 시 `YYYYMMDD` 입력만으로 `YYYY-MM-DD` 형식으로 자동 하이픈(-)이 삽입되며, 출생 시간 입력 시 `HHMM` 입력만으로 `HH:MM` 콜론(:)이 자동 마스킹되도록 개선하여 조작 편의성을 극대화했습니다.
*   **정보 수정/변경 페이지 연동**: 가입 완료(로컬스토리지 적재) 후 다시 온보딩에 진입하면 기존 저장 데이터가 인풋에 자동 바인딩되어 "정보 변경 모드"로 작동합니다. 변경 완료 시 전용 알림 창이 출력됩니다.
*   **프로필 초기화 (로그아웃)**: 기사 가입 테스트 편의성 및 데이터 파쇄를 위해 정보 변경 모드 하단에 **"프로필 정보 초기화 (로그아웃)"** 버튼을 구현했습니다.

### ② 사주 만세력 알고리즘 및 온보딩 유도 ([GillogPage](file:///d:/000_UNSU/fo/src/pages/GillogPage.tsx))
*   **미등록 예외 처리**: 프로필이 등록되지 않은 기사가 진입 시, 상단에 **"프로필 미등록 알림 배너"**를 골드 뱃지와 함께 노출하며 클릭 시 즉시 온보딩 페이지로 다이렉트 안내합니다.
*   **가독성 표기 개선**: 기사 프로필 연동 노출 시, 기존의 차가운 `"01-01"` 포맷 대신 따뜻하고 한국 정서에 부합하는 **`"01월 01일 생"`** 포맷으로 노출 형식을 수정했습니다.
*   **결정론적 일진 연산**: 기사 생일과 당일 날짜 문자열을 결합한 유니크 해시 수식을 빌드하여, 외부 유료 사주 API 호출 없이 기사 생일마다 매일 다르게 매핑되는 **"사주 기반 오늘의 행운 동선 내러티브"**를 동적으로 생성 렌더링합니다.

### ③ Web Speech API Native TTS 낭독 엔진 & ON DUTY 연동 ([GPanRadarPage](file:///d:/000_UNSU/fo/src/pages/GPanRadarPage.tsx))
*   **ON DUTY 상태 동기화**: 상단 공통 바의 `ON DUTY` 뱃지가 오디오 재생(`ON AIR`) 상태와 전역 컨텍스트로 동기화됩니다. 방송 청취 중에는 골드 글로우 맥동의 **ON DUTY**로 활성화되며, 정지(STANDBY) 시 차분한 그레이톤의 **OFF DUTY**로 즉시 전환됩니다.
*   **TTS 음질 개선 및 여성 음성 튜닝**: 브라우저별 Native TTS 목소리 중 `ko-KR` 여성을 우선 탐색(Yuna, Heami, Google 한국어 등)하여 음성에 할당합니다. 낭독 속도를 `0.90`배속으로 늦춰 부드러운 톤을 조성하고 피치를 `1.05`로 튜닝하여 인공지능 안내원 느낌을 대폭 개선했습니다.

### ④ 실시간 핫존 수동 업데이트 시스템 ([GPanRadarPage](file:///d:/000_UNSU/fo/src/pages/GPanRadarPage.tsx))
*   **수동 리프레시 및 최근 업데이트 시각**: 기사 광장에서 정보를 수동으로 갱신할 수 있도록 업데이트 버튼을 구축했습니다. 클릭 시 1초 동안 회전하는 애니메이션 로더 후, 현재 시간(예: `최근 업데이트 04:08:12`)을 뱃지 옆에 동적으로 마킹합니다.
*   **동적 데이터 갱신 시뮬레이션**: 수동 업데이트 시 강남역 사거리 및 김포공항의 대기 시간이 현실감 있게 무작위로 변경되어 실시간 업데이트 효능감을 제공합니다.

### ⑤ 영수증 OCR 리더보드 등극 및 모바일 카메라 촬영 ([RoadboarderPage](file:///d:/000_UNSU/fo/src/pages/RoadboarderPage.tsx))
*   **모바일 사진 직접 촬영 연동**: 영수증 파일 첨부 인풋에 모바일 표준 규격인 `capture="environment"` 속성을 부여하여 스마트폰 카메라 모듈을 통한 영수증 직접 촬영 인증을 지원합니다.
*   **OCR 파일 업로드 시뮬레이션**: 영수증 OCR 버튼을 누르고 이미지를 선택하면 **"이미지 압축 및 OCR 분석 중"** 바(Bar)가 1.5초간 작동하며 판독 상태를 연출합니다.
*   **실시간 랭킹 추가**: 판독 성공 즉시 온보딩된 기사의 이름(마스킹 아이디)과 분석된 매출액(54,200원)을 리스트에 삽입하고, 기존 랭킹 목록을 순차적으로 밀어내어 **1위로 즉시 실시간 랭킹 등극**하는 극적 인터랙션을 가동했습니다.
*   **LangSmith 허탕 피드백**: 핫존 허탕 아이콘 클릭 시, 런타임 수집 데이터가 LangSmith Tracing Dataset에 안전하게 축적되었음을 모달로 알려줍니다.

### ⑥ 사업자 맞춤형 간이과세 부가세 계산기 ([AutopilotPage](file:///d:/000_UNSU/fo/src/pages/AutopilotPage.tsx))
*   **개인택시 간이과세 감안 환급 연산**: 온보딩 시 기사가 `개인택시 (PRIVATE)`를 선택했을 경우, 적격 매입세액 85.6만 원에 간이과세 신용카드 등 공제 가중치 50%를 적용하여 **₩ 428,000** 환급액을 도출합니다.
*   `모범/대형 (PREMIUM)`인 경우 일반과세자로 분류해 매입세액 100%인 **₩ 856,000** 환급 결과를 실시간 노출하며 코멘트를 분기 처리합니다.

### 👉 테스트 가이드
로컬 서버 브라우저([Vite Dev App](http://localhost:5173))로 가셔서 아래 흐름으로 WOW 요소를 테스트해보세요!
1.  **G-PAN**: `ON AIR` 버튼을 눌러 스피커로 실제 한국어 인공지능 관제 멘트가 낭독되는지 확인합니다. 이와 동시 상단 바의 뱃지가 `ON DUTY`로 깜빡이는지 확인합니다.
2.  **수동 업데이트**: G-PAN 하단의 `업데이트` 버튼을 클릭하여 로딩 아이콘 회전 후 최근 업데이트 완료 시각 표시 및 대기 대기열 분 단위가 변하는지 검증합니다.
3.  **프로필**: 우측 상단 사람 아이콘을 눌러 생일과 홈택스 ID를 입력해 온보딩을 마칩니다. (숫자만 입력 시 자동으로 포맷팅 기호 `-`, `:`가 연동되어 들어갑니다)
4.  **정보 변경**: 가입 완료 후 다시 프로필 버튼을 누르면 기존 가입 정보가 불러와지며 정보 수정 모드로 작동함을 확인합니다. 하단 초기화를 누를 시 데이터가 파쇄되고 최초 상태로 돌아갑니다.
5.  **루틴**: 메인 홈으로 가면 미등록 알림 배너가 사라지고 기사님의 생일에 맞춘 사주 행운 코멘트로 갱신된 것을 확인합니다. 생일 표기가 `MM월 DD일` 생으로 포맷팅되었음을 확인합니다.
6.  **정산**: 오토파일럿 탭에서 정산을 돌려 기사 종류(간이/일반)에 따라 환급 계산액과 문구가 동적으로 달라지는지 검증합니다.
7.  **로드보더**: 영수증 OCR 인증을 누르고 카메라나 이미지를 첨부하여, 1.5초 로더 뒤 리더보드 1위로 실시간 랭킹 갱신이 일어나는지 모니터링합니다.
